# Methodology: First-Principles Thinking

This reference distills the thinking pattern from Li Shanyou's *第一性原理* into a practical coaching workflow.

## Core Frame

First-principles thinking asks: what is the base assumption outside or underneath this system, and what follows if we rebuild from it?

In this frame:

- Induction is useful for survival and ordinary operation, but it depends on continuity assumptions.
- Deduction can create a new system, but only if the base premise is strong enough.
- A first principle is not the center idea inside a system. It is the prior premise, base assumption, logical singularity, or "one" that supports the system.
- Every first principle has a scope. A conclusion in a parent system may become a first principle in a child system.
- Innovation often happens by breaking an old hidden assumption and installing a stronger base assumption.

## Distinguish Three Things

- **Surface content**: current tactics, features, plans, arguments, or solutions.
- **System center**: the main strategy, business model, product thesis, or organizing idea inside the system.
- **First principle**: the hidden base premise that makes the system center possible.

Do not mistake a slogan, KPI, strategy, product concept, or "main point" for a first principle. Ask what must already be true for that center idea to make sense.

## Why Induction Breaks

Induction generalizes from observed experience. It often smuggles in one of two assumptions:

- **Time continuity**: the future will be like the past.
- **Space continuity**: this context will behave like another context.

This works inside a stable boundary. It fails around non-continuity: market shifts, life transitions, technology changes, identity changes, strategy changes, or any move from one system to another.

When a conclusion looks wrong, do not only attack the conclusion. Look for the hidden continuity assumption beneath it.

## Break-And-Rebuild Pattern

Use this when the user wants a breakthrough, second curve, strategy reset, product reframing, or escape from inherited logic.

1. **Break hidden assumptions**
   - Identify what the current system treats as obvious.
   - Look for group beliefs, default industry logic, comparison thinking, past-success logic, identity protection, and untested constraints.
   - Ask what fails if the environment is no longer continuous with the past.

2. **Establish a stronger base assumption**
   - Propose a new premise outside or deeper than the current system.
   - Prefer principles from parent systems such as physics, biology, psychology, economics, sociology, mathematics, systems thinking, or philosophy when relevant.
   - The new premise must be stronger than the old one and usable in context.

3. **See the new system**
   - Deduce what follows from the new premise.
   - Let the new strategy, product, behavior, or decision emerge from the premise.
   - Remove elements that only make sense under the old assumption.

## First-Innovation Pattern

Use this when the user is building something new from a blank page.

1. Define the desired system and outcome.
2. Search for 2 to 4 parent-system principles that could govern it.
3. Combine a small number of strong principles into a working mental model.
4. Deduce the system design from those principles.
5. Validate with reality, treating cases and data as tests rather than sources of the whole model.

This moves from abstract to concrete. It differs from copying successful examples and generalizing from them.

## Useful Question Bank

System framing:

- What system are we actually examining?
- What is inside the boundary, and what sits outside it?
- What outcome is this system trying to produce?
- Where might the old boundary be breaking?

Hidden assumptions:

- What has to be true for the current approach to work?
- Which assumption is inherited from past success?
- Which assumption is inherited from another context?
- What does everyone believe here but rarely test?
- What would make this entire approach irrelevant?
- What would a thoughtful outsider see that insiders miss?

Finding the "one":

- What is the smallest premise from which this strategy follows?
- What is the premise behind the premise?
- Is this a base assumption, or merely a conclusion inside the system?
- Which parent system could provide a stronger principle?
- If we could only keep one governing idea, what would it be?

Rebuilding:

- If this premise is true, what must follow?
- What becomes unnecessary under the new premise?
- What becomes newly possible?
- What would a 10x better solution require us to stop assuming?
- What would the first cheap test be?

## Validation: Logic Three-Consistency

Treat any derived model as temporarily useful, not absolutely true.

- **Self-consistency**: the model does not contradict itself, and its claims follow from its premise.
- **Cross-system consistency**: the model does not violate stronger knowledge from adjacent or parent systems.
- **Continuity consistency**: the model can absorb new evidence, time shifts, and context changes without collapsing.

## Falsification Discipline

A useful first-principles model must be vulnerable to being proven wrong. Ask:

- What observation would make us abandon this premise?
- What result would show the derived strategy is not working?
- What cheap test would expose the biggest risk?
- Where might "we are right" or "everyone knows" be protecting the model from criticism?

If the model cannot be falsified, treat it as belief or rhetoric, not disciplined reasoning.
